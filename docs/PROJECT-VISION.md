## 5. Strategic Focus: Mixed-Attribute Expansion

### 5.1 The Core Design Principle

> **"Every agent can embody multiple attributes. Every feature serves multiple purposes. The mixed-attribute approach is not just a feature — it is the fundamental architecture."**

### 5.2 Expansion Strategy

| Direction | Description | Example |
|:----------|:------------|:--------|
| **Vertical Deepening** | Expand within existing categories | Add cardiologist, neurologist to medical |
| **Horizontal Expansion** | Add new categories | Add new professional fields |
| **Cross-Attribute** | Blend attributes in single agent | Entertainment-Professional hybrid |
| **Adaptive Integration** | Context-aware attribute switching | Professional → Entertainment based on context |

### 5.3 Mixed-Attribute Examples

| Agent | Primary | Secondary | Tertiary |
|:------|:--------|:----------|:---------|
| Chef Agent | Professional | Entertainment | Education |
| Code Architect | Professional | Functional | Education |
| Anime Character | Entertainment | Education | Professional |
| Language Tutor | Education | Professional | Cultural |
